Menu Maker by Qwenta
Mission
Ce projet s’inscrit dans le cadre du Projet 7 du parcours Intégrateur Web chez OpenClassrooms.
L’objectif est de planifier le développement d’un site web appelé Menu Maker, destiné aux restaurateurs, afin de leur permettre de créer facilement et rapidement leurs menus personnalisés.

En tant que développeur front-end au sein de l’agence Webgencia, vous êtes chargé de piloter la phase préparatoire du projet en l’absence du Product Owner.

Le projet inclut principalement :

Une veille technologique pour rester informé des dernières tendances et outils pertinents au projet.

La définition précise des spécifications techniques basées sur les maquettes et besoins fonctionnels.

La création d’un tableau Kanban structuré suivant une méthodologie agile pour organiser et suivre les tâches de développement.

La rédaction d’un rapport complet et d’une présentation PowerPoint à destination du client.

Réalisations
Veille technologique
Organisation et gestion de la veille via la plateforme Pearltrees.

Thèmes abordés : React, Node.js, API REST, bases SQL/NoSQL, accessibilité, sécurité, bonnes pratiques.

Document de veille complet, synthétisant les sources d’informations et leur impact sur le projet.

Consultez la veille ici (PDF) : 1_veille_technologique.pdf
Lien Pearltrees pour la veille : Lien de la veille

Spécifications techniques
Élaboration des spécifications techniques en se basant sur la maquette officielle, les spécifications fonctionnelles, et les user stories.

Description détaillée de l’architecture technique, des choix technologiques, des fonctionnalités principales et des contraintes.

Consultez les spécifications complètes (PDF) : 2_spécifications_techniques.pdf

Gestion du projet avec un Kanban
Création d’un tableau Kanban sur Trello suivant les bonnes pratiques agiles.

Répartition des tâches avec descriptions claires, assignation, estimation en story points, état d’avancement.

Suivi visuel et priorisation du travail en cours.
